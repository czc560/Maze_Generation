# 算法复杂度与效果分析

## DFS 回溯法

时间复杂度约为 `O(V)`，其中 `V` 是奇数网格节点数。空间复杂度为递归/栈深度 `O(V)`。生成迷宫通常通路蜿蜒、死胡同较多，适合展示“探索与回溯”。

## Prim 算法

随机 Prim 使用 frontier 集合扩展迷宫，时间复杂度约为 `O(V + E)`。因为本项目用列表随机取 frontier，在大规模迷宫上会有额外常数开销。生成结果分支较均衡，适合 AI 寻路挑战。

## Kruskal 算法

Kruskal 将可通行格视为图节点，将墙视为边，使用并查集判环。时间复杂度约为 `O(E α(V))`，其中 `α` 为反 Ackermann 函数。生成结果是树型完美迷宫，便于证明唯一通路。

## 递归分割法

递归分割法从全通路开始不断加墙并开门。平均时间复杂度约为 `O(n^2)`。该算法通常产生更规整的房间/走廊结构，不强制完美迷宫，但保证连通。

## BFS / 分支限界优化

优化器先计算路径、分支、死胡同等指标，再尝试打开局部墙体。每次候选修改都调用验证器，单次验证复杂度约 `O(n^2)`。优化目标是提升分支与挑战性，同时不破坏合法性。

## 对比指标

项目输出以下指标：

- 运行时间；
- 空间估计；
- 墙数量；
- 通路数量；
- 死胡同数量；
- 分支数量；
- 最短路径长度；
- 是否完美迷宫；
- AI 贪婪策略得分；
- 资源最优路径得分。

运行：

```bash
python run.py --compare --size 15 --seed 42
```
